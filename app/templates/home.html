Application de déploiement de modèles ML

Travail effectué

  1. Déploiement d’un modèle de reconnaissance d’objet VGG16 en utilisant FastAPI. Le point d’entrée pour ce modèle est /predict.
  2. Sécurisation du code pour réduire le risque d’upload de fichiers à risque, en utilisant la bibliothèque werkzeug.utils.
  3. Création d’un conteneur Docker pour cette application. Un volume est utilisé pour sauvegarder les fichiers téléchargés sur la machine hôte.
  4. Création d’un second endpoint pour déployer un modèle ML sous forme Pickle. Le point d’entrée est /regpredict.
  5. Création d’un troisième endpoint /textgen pour déployer un modèle Hugging Face (GPT2).
  6. Création de ce endpoint /home pour présenter et expliquer le projet, avec les étapes et commandes pour déployer automatiquement l’application.

Déploiement

On utilise FastAPI comme un seul service (est donc pas besoin de docker-compose, car on a un seul Dockerfile), le déploiement se fait avec :

# Construire l'image Docker
docker build -t ml-deploy:latest .

# Lancer le conteneur avec volume
docker run -p 3000:3000 -v ../uploads:/app/uploads ml-deploy:latest

Les endpoints disponibles sont :

  • /predict → modèle VGG16
  • /regpredict → modèle de régression Pickle
  • /textgen → génération de texte GPT2
  • /home → cette page
